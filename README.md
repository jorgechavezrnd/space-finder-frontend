# Space Finder Frontend

Frontend application for the **Space Finder** project — a web app that allows users to find and manage spaces/locations. This app is built to be deployed on **AWS**, integrating with the Space Finder backend services provisioned via AWS CDK.

## Project Initialization

This project was scaffolded using [Vite](https://vite.dev/) with the React + TypeScript template:

```bash
npm create vite@latest . -- --template react-ts
```

## Deployment

### 1. Build the project

From the `space-finder-frontend` folder, run:

```bash
npm run build
```

### 2. Deploy to AWS

From the [`space-finder`](https://github.com/jorgechavezrnd/space-finder) folder, run:

```bash
cdk deploy UIDeploymentStack
```

### 3. Invalidate the CloudFront cache

CloudFront caches the content for up to **24 hours**, so changes won't be visible immediately after deployment. To force the cache to refresh, run:

```bash
aws cloudfront create-invalidation --distribution-id E3125AYEMGM8VV --paths "/*"
```

> The distribution ID (`E3125AYEMGM8VV`) can be found in the AWS Console under:
> **CloudFormation → UIDeploymentStack → Resources → SpaceFinderDistribution → SpacesFinderDistribution3DAEE455 → Physical ID**
>
> ⚠️ Note: the `3DAEE455` suffix is dynamically generated by CDK and may differ across deployments.
